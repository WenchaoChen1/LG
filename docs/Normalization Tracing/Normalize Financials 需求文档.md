# 产品需求文档：财务数据标准化系统

## 一、功能概述

Normalize Financials 是一个财务数据标准化平台，支持 Looking Glass 将QuickBooks、Financial Entry的财务数据转换为标准化、符合GAAP(General Accetped Accounting Principle)的统一结构。

### 核心目标
- 标准化多源异构财务输入，实现跨公司对标
- 自动化指标表映射，减少手动对账时间
- 提升数据可信度，支持投资者、运营商决策
- 实现可审计、可追踪的数据处理
- 支持可扩展的客户加入，消除财务团队瓶颈

---

## 二、功能结构

### 2.1 核心功能模块

系统包含 **4 个独立但相关的功能模块**：

| 功能 | 描述 |
|------|------|
| **Normalization Engine** | 将原始财务数据转换为标准化、可比较的分析就绪输出 |
| **Mapping Logic** | 将原始指标行项目映射到标准化类别，保留完整追踪 |
| **Nightly Normalization** | 自动定时处理新增/更新的财务数据，创建版本化输出 |
| **Normalization Tracing** | 追踪标准化数据回源到原始输入 |

## 三、业务逻辑

### 3.1 规范化引擎（Normalization Engine）

#### 工作原理
系统采用 **通用核心逻辑 + 规则集配置** 的设计：
- 核心引擎逻辑在所有公司间统一共享
- 每个公司分配一个专属的规范化规则集（Ruleset）
- 规则集定义公司特定的配置项

#### 规则集配置项
规则集必须定义：
1. **货币转换** - 使用哪个月度汇率进行转换
2. **收入展示方法** - 支持三种方式：
   - Last Month Method：ARR = 最近一个月MRR × 12
   - 3-Month Average Method：ARR = 最近3个月MRR平均值 × 12
   - Trailing Twelve Months (TTM)：ARR = 过去12个月收入总和
3. **会计基础属性** - 现金制或应计制（仅作为描述属性，不做转换）

**MVP不包含**：财政日期对齐规则、会计基础转换逻辑（架构预留扩展空间）

#### 执行流程

**Step 1：检索激活的规范化规则集**
- 根据公司标识加载其活跃规则集
- 验证规则集版本

**Step 2：执行核心财务转换**

**A) 货币转换**
- 所有货币值转换为 USD，使用存储的月度汇率
- 如缺少汇率：
  - 记录验证警告
  - 标记受影响的记录
  - 保留原始币种值（不转换）

**B) 收入展示调整**
根据规则集选择的方法计算 ARR：
- Last Month：ARR = 最近月份 MRR × 12
- 3-Month Average：ARR = 最近3个月平均 MRR × 12（不足3个月则警告后用现有数据）
- TTM：ARR = 过去12个月总收入（不足12个月则警告后用现有数据）

**C) 会计基础处理**
- **MVP不做转换**：保留公司报告的原始基础（现金或应计）
- 会计基础作为元数据存储，用于下游系统（Benchmark）的同类对标
- 架构设计允许未来阶段添加转换规则

**Step 3：自动计算衍生指标(需要计算的指标）**
系统使用标准公式自动计算所有衍生指标，应用于标准化数据：
- 基础：Gross Profit、Net Income 等
- 高级：Rule of 40、Sales Efficiency Ratio、Monthly Runway 等
- 衍生指标应一致、持久化存储、作为单一真实来源

---

### 3.2 数据映射逻辑（Mapping of Raw Data）

#### 映射规则
系统基于规则将原始指标行项目映射到标准化类别：

**标准映射关系表**：
| 原始数据（LG） | 目标类别 |
|---|---|
| Gross Revenue | Revenue |
| COGS | COGS |
| S&M Expenses | OPEX |
| S&M Payroll | OPEX |
| R&D Expenses | OPEX |
| R&D Payroll | OPEX |
| G&A Expenses | OPEX |
| G&A Payroll | OPEX |
| Misc OpEx | OPEX |
| 其他行项目 | 各自独立行项目 |

#### 映射存储与追踪
每条标准化记录必须存储：
- 原始源记录 ID
- 原始指标名称与金额
- 分配的标准化类别
- 使用的映射规则或逻辑
- 映射版本标识符

#### 映射版本管理
- 映射逻辑版本化管理
- 映射规则变更时：
  - 新的规范化运行使用更新的映射逻辑
  - 历史标准化数据保持不变
  - 系统支持使用历史版本规则重新规范化

---

### 3.3 会计方法设置（Cash/Accrual Setting）

#### 配置说明
每个公司在 **Company Settings > Finance 标签页** 必须有一项会计方法设置：
- 选项：Cash 或 Accrual（默认）
- 存储位置：公司级别
- 属性：纯元数据，不作为转换规则

#### 规范化行为
- 规范化引擎 **不执行** 现金/应计转换
- 规范化 **不改变** 基于会计方法的财务数值
- 会计方法仅用于下游用途（Benchmark中的同类分组）
- 会计方法记录为规范化运行元数据（审计可追踪）

#### 基准测试与变更管理
- 基准测试逻辑使用会计方法进行同类分组（现金公司仅与现金公司对标）
- 修改会计方法 **不影响** 历史标准化数据
- 修改会计方法 **不触发** 历史数据重新规范化
- 修改仅影响未来规范化和基准测试

### 3.4 夜间规范化存储过程（Normalization Stored Procedure）

#### 触发机制
自动每晚执行，检测自上次成功规范化运行以来的原始财务数据变化：
- 手动数据输入
- QuickBooks 同步更新
- 先前提交财务的修正

#### 数据稳定性判断
规范化仅在数据稳定/关闭时执行：
- **QB公司**：每月15日之后
- **手动输入公司**：数据输入/更新后立即执行

#### 处理流程
1. 识别有新增或更新原始数据的公司
2. 仅对检测到变化的公司执行规范化（无变化则跳过）
3. 对每个符合条件的公司：
   - 应用当前规范化规则集
   - 使用公司的财务设置（财年开始月、会计方法、货币处理）
   - 生成完整的月度标准化数据集

#### 版本管理与数据保留
- 每次成功规范化创建新版本
- 历史标准化数据集保持不变且可访问
- **永不覆盖或删除** 规范化数据
- 每条规范化记录关联：
  - 规范化版本
  - 规则集版本
  - 执行时间戳

#### 错误处理
- 一个公司的失败 **不阻止** 其他公司处理
- 公司规范化无法完成时：
  - 记录清晰的失败原因
  - 之前的规范化数据集保持活跃
- 部分数据问题（缺汇率等）：
  - 带警告完成处理
  - 记录验证消息

#### 日志与可见性
每次夜间运行记录：
- 处理的公司清单
- 跳过的公司清单
- 每个公司状态：Success / Success with Warnings / Failed
- 验证警告和错误，关联到规范化版本
- 每个公司最近成功规范化可识别

#### 下游消费
规范化输出立即可用于：
- 基准测试
- 投资组合分析
- 无需额外转换

### 3.5 规范化溯源（Normalization Tracing）

#### 访问权限
- **仅行政用户** 可访问
- 行政用户仅可见其有权限的活跃公司

#### 操作流程（分级钻取）
用户按以下步骤选择：
1. 选择公司
2. 选择财务周期（月份）
3. 选择标准化指标或类别

#### 源系统确定逻辑
源系统在 **公司-月份级别** 确定，而非每个指标单独确定。

根据摄入历史识别唯一权威源系统：
- **仅手动输入** → 所有月份源系统 = Manual Entry
- **仅QB** → 所有月份源系统 = QuickBooks API
- **QB → 手动切换** → 所有月份源系统 = Manual Entry
- **手动 → QB 切换（启用覆盖）** → 所有月份源系统 = QuickBooks API
- **手动 → QB 切换（未启用覆盖）**：
  - 历史月份 + 当前月份 → Manual Entry
  - 未来月份 → QuickBooks API

#### 溯源结果展示

**页面分为三个清晰分区**：

**1. 标准化输出**
- 标准化指标名称
- 标准化数值（USD）
- 所选月份的已解析源系统（突出显示）
  - 示例："Source System (for selected month): Manual Entry"

**2. 原始源数据**
- 原始指标/类别名称
  - 若非1对1关系，显示 "Aggregated from multiple sources"
- 原始金额
  - 若非1对1关系，显示 "-"
- 源系统（继承自月份级别确定）
- 按升序排列（主排序键：Company，次排序键：Metric Name）
- "More Details" 按钮：查看完整信息

**3. 规范化与计算逻辑**
- **科目映射**："Mapped to Normalized Category"
- **货币转换详情**：
  - 原始货币
  - USD数值
  - 使用的汇率
  - 转换日期
- **衍生计算逻辑**：
  - 显示公式以追踪标准化输出来源
  - 信息提示：说明指标来自规范化表或LG FE表
  - 多个公式时在弹窗展示

#### 数据特性
- 所有数据为 **只读**
- 数据反映所选周期的 **当前活跃规范化版本**
- 溯源输出与下游基准测试和分析使用的数据 **保持一致**

#### 用户界面设计
- 位置：管理员后台导航右侧面板
- 标签：如 "Normalization Trace" 或 "Data Lineage"
- 使用卡片、面板或分隔线区分各部分
- 预留空间支持未来添加新源系统和补充规则

## 四、数据模型

### 4.1 标准化财务类别（必须）
- **Revenue**（收入）
- **COGS**（成本）
- **Gross Margin**（毛利）
- **Operating Expenses (OPEX)**（运营费用）
- **EBITDA**
- **Net Income**（净收入）
- 其他类别可无需重设计即可扩展

### 4.2 衍生指标（完整列表）
系统必须支持计算并持久化以下指标：

**盈利能力类**
- Gross Profit
- Gross Margin
- Net Profit Margin
- EBITDA

**收入类**
- ARR（Annual Recurring Revenue）
- MRR（Monthly Recurring Revenue）
- New MRR LTM
- MRR YoY Growth Rate

**效率类**
- Sales Efficiency Ratio
- Rule of 40

**现金流类**
- Monthly Net Burn Rate
- Monthly Runway
- Runway Left
- End of Runway

**资产负债类**
- Assets
- Liabilities
- Equity
- Debt/Asset Ratio

**其他**
- Capitalized R&D (Total)
- Miscellaneous Operating Expenses

### 4.3 数据粒度与版本
- **时间粒度**：月度（Monthly Granularity）
- **版本管理**：每次规范化创建新版本，历史数据保持不变
- **数据状态**：Raw → Mapped → Normalized → Approved
- **追踪信息**：规范化版本、规则集版本、执行时间戳

---

## 五、完整可追踪性需求

对于每条标准化财务记录，系统必须追踪：
1. **源数据** - 原始原始记录
2. **映射关系** - 应用的科目映射或类别分配
3. **规则版本** - 使用的规则或规则集版本

每条标准化记录必须包含：
- 公司标识
- 月度周期
- 标准化财务类别
- 标准化金额
- 规范化规则版本
- 回源的原始数据信息

## 六、接受标准（Acceptance Criteria）

### 规范化数据存储
- ✓ 标准化财务包含所有必需类别（Revenue、COGS、OPEX等）
- ✓ 每条记录可识别：公司、月份、类别、金额、规则版本、原始数据追踪
- ✓ 任何记录可追踪回：精确的原始值、应用的映射、使用的规则版本

### 规范化引擎
- ✓ 正确加载每个公司的活跃规则集
- ✓ 所有货币值转换为USD，缺汇率则警告且保留原值
- ✓ ARR按公司选定方法计算并持久化
- ✓ 不执行会计基础转换，保留为元数据
- ✓ 所有现有LG指标都有衍生指标计算
- ✓ 规范化版本唯一，历史数据可访问

### 映射逻辑
- ✓ 原始行项目映射到标准类别（Revenue、COGS、OPEX或独立行项目）
- ✓ 每条记录存储：源ID、原始科目、标准化类别、映射规则、版本
- ✓ 映射逻辑版本化，规则变更时新运行使用新规则，历史数据不变
- ✓ 系统支持使用历史版本重新规范化

### 会计方法
- ✓ 每个公司有 Cash/Accrual 设置（默认 Accrual）
- ✓ 规范化不基于会计方法修改数值
- ✓ 会计方法用于基准测试的同类分组
- ✓ 修改会计方法不改变历史数据

### 夜间规范化
- ✓ 自动定时执行，检测变化的公司
- ✓ 应用当前规则集和公司财务设置
- ✓ 按月粒度生成标准化数据
- ✓ 创建新版本，保留历史版本
- ✓ 一个公司失败不阻止其他公司
- ✓ 记录：处理/跳过的公司、状态、验证消息
- ✓ 输出立即可供基准测试和分析使用

### 规范化溯源
- ✓ 仅管理员可访问，仅见活跃公司
- ✓ 支持选择：公司、月份、指标
- ✓ 确定月份级别的权威源系统
- ✓ 显示：标准化值、源系统、原始行项目（含科目名、金额）
- ✓ 显示：映射、汇率、转换日期、衍生公式
- ✓ 所有数据只读，反映当前活跃版本
