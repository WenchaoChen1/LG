# Looking Glass 预测数据标准化 - 需求文档

## 一、概述

扩展 Looking Glass 现有的标准化框架，使其支持预测财务数据（承诺预测和系统生成预测）的处理，使标准化的预测数据可用于预测基准对标分析。

---

## 二、需求范围

### 2.1 支持的预测数据源
- **承诺预测（Committed Forecast）**：用户提交的财务预测
- **系统生成预测（System-Generated Forecast）**：算法引擎自动生成的预测

两种预测数据应独立处理，每个财政年度使用最新可用的预测版本进行完整的数据集标准化。

### 2.2 核心功能需求

#### (1) 复用现有标准化引擎
- 使用同一套规则引擎处理预测数据和历史数据
- 运行时获取并应用公司有效规则集，该规则集定义了以下公司特定配置：
货币转换
收入列报方式（上月、三个月平均值或TTM）
会计基础（现金制或权责发生制），仅作为描述性属性
用于审计和版本控制的元数据
- 无需创建预测专用的重复标准化逻辑

#### (2) 科目(Metric)映射与追溯
- 预测原始财务科目映射到标准科目体系（Revenue、COGS等）
- 使用现有规则映射逻辑
- 保持映射追溯记录

#### (3) 核心转换
将对历史数据的所有转换应用于预测数据：
- **货币转换**：使用月度汇率转换为USD
- **收入展示标准化**：支持Last Month、3-Month Average、TTM等多种展示维度
- **衍生指标计算**：自动计算Gross Profit、EBITDA等衍生指标

#### (4) 数据隔离
- 预测标准化结果独立存储（与历史表分离）
- 标准化过程不修改或影响历史数据

#### (5) 版本管理
每次标准化运行生成唯一版本，版本信息包含：
- 公司ID
- 预测类型（Committed或System-Generated）
- 应用的规则集版本
- 处理时间戳

版本仅在数据实际发生变化时生成；

#### (6) 审计与可追溯性
- 记录每次标准化操作的完整日志
- 保存版本历史供查询和对比
- 支持版本追溯和恢复

#### (7) 重新处理机制
支持以下场景的自动或手动重新处理：

| 触发条件 | 处理方式 |
|---------|--------|
| 已承诺预测被编辑或接受 | 自动重新处理 |
| 系统生成预测被刷新 | 自动重新处理 |
| 公司规则集变更 | 自动重新处理所有相关预测 |

新的重新处理运行仅在检测到数据变化时才生成新版本。

#### (8) 基准对标就绪
- 标准化预测指标持久化存储
- 预测指标可直接用于基准对标计算
- 支持预测与历史的基准对标比较

---

## 三、数据流

### 3.1 输入数据
- 预测原始数据（科目、金额、货币、期间）
- 公司规则集配置
- 月度汇率表

### 3.2 处理流程
```
预测数据
  → 加载公司规则集
  → 科目映射
  → 货币转换
  → 收入展示标准化
  → 衍生指标计算
  → 版本生成和记录
  → 标准化结果存储
```

### 3.3 输出结果
- 标准化的预测指标集（按标准科目和衍生指标）
- 版本元数据（公司、预测类型、规则版本、处理时间等）
- 日志记录

## 四、关键需求项

| 需求项 | 说明 |
|-------|------|
| **完整数据集** | 每个财政年度必须生成完整的预测标准化数据集|
| **规则一致性** | 预测和历史数据必须使用同一规则引擎和映射逻辑 |
| **数据独立性** | 预测和历史标准化表必须分离，互不影响 |
| **版本清晰** | 版本号唯一标识一次标准化运行，记录所有必要元数据 |
| **自动化处理** | 数据变更和规则变更时自动触发重新处理 |
| **可审计性** | 保持完整的操作日志和版本历史 |
| **对标支持** | 标准化预测数据可直接用于预测基准对标计算 |

---

## 五、非函数需求

- 不修改现有标准化规则引擎的核心逻辑
- 保持与现有历史数据处理的一致性
- 支持版本回滚和历史查询
