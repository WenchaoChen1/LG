# Forecast数据标准化追踪功能需求文档

## 一、功能概述

本功能是对现有 **Normalization Tracing（标准化追踪）** 功能的扩展，用于支持预测数据的标准化过程追踪。

**当前状态**：Normalization Tracing 页面已支持 Historical Actuals（实际财务数据）的追踪

**功能扩展**：新增对 Committed Forecast（承诺预测）和 System Generated Forecast（系统生成预测）的标准化追踪能力

**核心目标**：使 Admin 用户能够追踪预测衍生的标准化指标的完整来源链路，包括输入值、转换逻辑和数据类型

---

## 二、业务需求

1. **支持多数据类型追踪**：能够区分和追踪三种数据类型的标准化值
   - Historical Actuals（实际财务数据）
   - Committed Forecast（承诺预测）
   - System Generated Forecast（系统生成预测）

2. **预测数据标准化可追踪性**：预测衍生的标准化指标必须可追踪，使 Admin 用户能够验证其正确性

3. **基准对标数据验证**：确保追踪视图中的预测标准化值与基准引擎使用的标准化值一致

4. **保持追踪原则**：遵循现有追踪框架的设计原则
   - 数据来源可见性
   - 转换逻辑透明
   - 读写审计功能
   - Admin-only 访问权限

---

## 三、操作流程

### 3.1 追踪流程（多步骤）

| 步骤 | 操作名称 | 用户操作 | 系统反馈 |
|------|--------|--------|--------|
| 1 | 选择公司 | Admin 选择目标公司 | 确认选择 |
| 2 | 选择财务期间 | 选择特定月份 | 确认选择 |
| 3 | **选择数据类型** | 在三种数据类型中选择 | 显示对应数据可用性 |
| 4 | 选择标准化指标/类别 | 选择要追踪的指标 | 显示追踪结果 |

**数据类型选择项**：
- Historical Actuals（实际数据）
- Committed Forecast（承诺预测）
- System Generated Forecast（系统生成预测）

## 四、操作反馈

### 4.1 用户触发操作的反馈

| 操作 | 即时反馈 | 延迟/后续反馈 |
|------|--------|------------|
| 选择数据类型 | 页面刷新，显示对应数据类型的可用指标列表 | — |
| 选择指标/类别 | 立即展示追踪结果视图 | — |
| 查看追踪详情 | 展示完整的标准化链路 | — |

### 4.2 系统状态反馈

| 场景 | 反馈形式 | 内容 |
|------|--------|------|
| 数据类型选择后 | 标识显示 | 清晰标明当前选择的数据类型（Historical / CF / SF） |
| 预测数据不可用 | 禁用状态或提示 | 若该月份/公司无该类型预测数据，显示"无可用数据" |
| 追踪完整性 | 验证标记 | 确认显示的标准化值与基准引擎使用值一致 |
| 数据时效性 | 版本标识 | 显示使用的活跃标准化版本 |

## 五、页面描述

### 5.1 Normalization Tracing 页面（扩展版）

**现有结构保留**：
- 第1步：选择公司
- 第2步：选择财务期间
- 第3步：**新增 - 选择数据类型**（可多选）
- 第4步：选择标准化指标或类别
- 第5步：列表Source栏新增两种来源Committed Forecast、System Generated Forecast

### 5.2 Data Type选择器（新增）

**UI 形式**： 下拉选择框

**选项**：
- ○ Historical Actuals（实际财务数据）
- ○ Committed Forecast（承诺预测）
- ○ System Generated Forecast（系统生成预测）

**属性**：
- 必填项：是
- 影响范围：影响第4步展示的可用指标列表

### 5.3 Metric Detail页面
- Source数据来源包含以下四种：
Manual Entry：手动输入的真实数据
QuickBooks API：自动拉取的真实数据
Committed Forecast：Committed Forecast数据(新增)
System Generated Forecast：系统生成的预测数据(新增)
- OPEX Calculation Logic改为： OPEX = OPEX * FX Rate

## 六、数据来源与存储

| 数据项 | 来源系统 | 存储方式 | 更新频率 |
|------|--------|--------|--------|
| 承诺预测（Committed Forecast） | 用户在平台手动输入 | Forecast 数据库，关联公司和期间 | 用户维护 |
| 系统生成预测（System Generated Forecast） | 系统预测模型 | Forecast 数据库，关联公司和期间 | 模型定期生成 |
| 标准化规则 | 配置管理系统 | 版本化存储，标记活跃版本 | 管理员更新 |
| 实际财务数据（Historical Actuals） | 手动录入 / QuickBooks| 财务数据库 | 月度或实时 |
| 货币汇率 | 外部数据源 / 手动维护 | 汇率查询表 | 日度或月度 |
| 追踪审计日志 | 系统生成 | 审计日志表 | 实时记录 |

## 七、业务逻辑闭环

### 完整追踪链路

```
Admin 用户访问 Normalization Tracing 页面
    ↓
选择公司 → 选择期间 → 【新增】选择数据类型 → 选择指标
    ↓
系统识别数据类型（Historical / CF / SF）
    ↓
根据数据类型获取对应的原始数据
    ├─ Historical：从财务数据库读取
    ├─ CF：从预测数据库读取用户输入值
    └─ SF：从预测数据库读取模型生成值
    ↓
应用标准化规则（相同规则，不同数据来源）
    ├─ 类别映射
    ├─ 货币转换
    └─ 衍生计算
    ↓
生成标准化值
    ↓
与基准引擎对标，验证一致性
    ↓
展示完整追踪链路给 Admin 用户
    ↓
Admin 获得完整的数据来源和转换可见性
```
