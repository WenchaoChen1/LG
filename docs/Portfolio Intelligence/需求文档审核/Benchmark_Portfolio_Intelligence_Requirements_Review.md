## 功能理解

Portfolio Intelligence Benchmarking 是 Portfolio Companies 模块下新增的 Tab 页，为管理员提供**多公司级别的财务指标对标分析**。用户先通过 Companies 下拉框选择要对标的公司，然后通过 VIEW/FILTER/DATA/BENCHMARK 四组筛选器配置分析维度，最后在两种视图中查看结果：

- **Snapshot（快照）**：选定单月，以表格形式展示每家公司在各指标上的百分位排名和实际值，支持分页（每页最多 10 家公司）。每行按 DATA 类型分子行（黑色=Actuals、紫色=Committed Forecast、紫色带标记=System Generated Forecast），每列按 FILTER 选中的指标动态增减。
- **Trend（趋势）**：选定月份区间，按 6 个指标分为独立卡片（默认展开，可收起），每个卡片内按 DATA×BENCHMARK 组合生成折线图，每条折线代表一家公司的百分位排名变化。

核心数据流：用户选公司+筛选器 → 前端请求后端（传公司ID列表、DATA/BENCHMARK/日期参数）→ 后端返回各公司的百分位+实际值 → 前端按视图类型渲染表格或折线图。

关键概念关系：FILTER 控制展示哪些指标列/卡片（Growth→ARR Growth Rate, Efficiency→Gross Margin 等）；DATA 和 BENCHMARK 控制"用什么数据"和"跟谁比"，它们的笛卡尔积决定表格行内子行数量和 Trend 模式下的折线图数量。

## 🔴 阻断性问题（共 4 个）

> 逻辑矛盾、交互规则缺失、公式错误 —— 不修复则无法开工

### 按章节分组

#### §筛选条件区域（2 个）

1. **Data 筛选器最少选择数量未定义**
   - 位置：§4 Data 筛选器（行 141）
   - 原文：「交互方式：多选标签,默认Actuals」
   - 问题：新版文档补充了默认值描述，但仍未定义**能否全部取消选择**。如果用户依次取消所有 DATA 选项，页面应该怎么办？FI 模块的实现是**阻止取消最后一项**（至少保留一个），本文档未明确。这直接影响前端交互逻辑。
   - 建议：在限制中补充"至少选择一项数据类型，不允许全部取消；若用户尝试取消最后一项，操作无效"。

2. **Benchmark 筛选器最少选择数量未定义 + 控件类型与 UI 矛盾**
   - 位置：§5 Benchmark 筛选器（行 161）
   - 原文：「交互方式：多选复选框，默认Internal Peers」
   - 问题：(1) 同 DATA 筛选器，未定义最少选择约束。(2) 文档写"多选复选框"，DATA 写"多选标签"，两者交互方式不同（复选框有勾选框 UI，标签是 pill 按钮 UI），但从 UI 截图看两者都是金色 pill 按钮样式，术语与 UI 矛盾。
   - 建议：(1) 统一术语为"多选标签（pill button）"；(2) 补充最少选择一项的约束。

#### §快照视图（1 个）

3. **Overall Benchmark Score 的计算公式与多维度展示矛盾**
   - 位置：§快照视图 - 表格列（行 209）
   - 原文：「Overall benchmark score:显示为例如：35%ile，数据值为该行中4-N列所有列的百分位的算术平均值，百分位是N/A时不参与计算」
   - 问题：每列在不同 DATA×BENCHMARK 组合下有不同百分位值（如 ARR Growth Rate 在 Actuals-KeyBanc 下是 P56，在 CommittedForecast-KeyBanc 下是 P42）。**Overall Score 按哪个维度计算？** 从 UI 截图看，Overall Score 列也显示了多行（对应多个 DATA×BENCHMARK 组合），应该是**每个 DATA×BENCHMARK 组合分别计算一个 Overall Score**，但文档"该行中4-N列所有列的百分位"未区分维度。
   - 建议：改为"Overall Benchmark Score 按 DATA×BENCHMARK 维度分别计算。每个维度的 Overall Score = 该维度下所有可见指标列（4-N列）的百分位算术平均值（N/A 不参与计算且不计入分母），四舍五入取整 + `%ile` 后缀"。

#### §趋势视图（1 个）

4. **数据缺失时展示 P0 仍有视觉误导**
   - 位置：§趋势视图 - 数据缺失处理（行 356）
   - 原文：「某家公司某个月份无数据，则展示最低点P0，tooltip展示N/A」
   - 问题：新版补充了"tooltip展示N/A"（改进），但核心视觉问题仍在——P0 表示"排名最差"而非"没有数据"。多公司对比中，某月无数据的公司折线跌至 P0，视觉上会被误读为"该月表现极差"。用户不会每次都悬停 Tooltip 确认。
   - 建议：改为以下方案之一：(1) 无数据月份折线断开（`connectNulls: false`），数据点不显示；(2) 使用虚线连接缺失段，数据点用空心圆标记，Tooltip 标注"(No Data)"；(3) 如果业务确实要求画在 P0，则在该数据点用特殊标记（如空心 × 号）区分于正常数据点。

#### §筛选条件区域（3 个）

2. **Filter 筛选器多选行为边界未定义**
   - 位置：§3 Filter 筛选器（行 120-124）
   - 原文：「交互方式：多选标签,默认All」+「"All" 与其他分类选项互斥」
   - 问题：互斥规则描述了"选 ALL 时其他取消"和"选非 ALL 时 ALL 取消"两个方向，但未覆盖以下边界：(1) 用户能否同时选 GROWTH + EFFICIENCY？（可以推断为"是"，但未明确说明）(2) 如果已选 GROWTH + EFFICIENCY，取消 EFFICIENCY 后剩 GROWTH 一个，是否允许？(3) 如果只剩一个非 ALL 项时取消它，是回到 ALL 还是允许空选？

3. **Snapshot 默认日期"当前月份"定义模糊**
   - 位置：§6 日期选择器（行 189-191）
   - 原文：「Snapshot：当前月份」「Trend：默认6个月，当前月及当前月前数5个月」

   - 建议：改为"默认选择最近的已关账月份（closed month），即当前日历月的上一个月"，或由 API 返回 `defaultDate`。

4. **日期选择器可选范围及校验规则未定义**
   - 位置：§6 日期选择器（行 186-191）

   - 建议：定义可选范围（建议最近 24 个月，不允许选未来）；Trend 的 start 可选范围自动 < end，end 可选范围自动 > start；展示格式建议 "YYYY - MM"。

   - 位置：§快照视图 - 计算逻辑（行 243-251，新增内容）

   - 建议：在表头补充说明——"方向：百分位计算时的排序方向。升序=值越大百分位越高（higher is better）；降序=值越小百分位越高（lower is better）"。

5. **Peer Fallback 提示在 Portfolio 级别的展示方式未定义**
