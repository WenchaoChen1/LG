# Benchmark Entry 产品需求文档（PRD）

> 版本：v1.0 | 更新日期：2026-03-19

---

## 一、功能目的

Benchmark Entry 是面向投资分析团队的 SaaS 行业基准指标管理工具，解决以下核心问题：

| 问题 | 解决方案 |
|------|----------|
| 基准数据分散在 Benchmarkit.ai、KeyBanc、High Alpha 等多个平台，查找对比低效 | 提供统一界面集中管理所有平台的基准数据 |
| 各平台指标口径不同，无法直接横向对比 | 通过 Looking Glass（LG）指标体系对各平台指标进行统一归类映射 |
| 数据录入涉及多字段，传统表单效率低 | 采用表格内联编辑模式，所有操作在同一页面完成，零页面跳转 |
| 投资决策需要多维度参考数据 | 支持按平台、年度、分位值等维度组织数据，辅助投资判断 |

## 二、使用场景

### 场景 1：季度基准数据录入

| 项目 | 说明 |
|------|------|
| **角色** | 投资分析师 |
| **触发条件** | 每季度各数据平台发布新的 SaaS 行业基准报告 |
| **目标** | 将最新的基准数据录入系统，保持数据时效性 |

**操作路径**：
1. 进入 Benchmark Entry 页面
2. 点击 "ARR Growth Rate" 展开详情区域
3. 点击 "Add detail"，新增一行空白行
4. 依次填写：Platform 选择 "Benchmarkit.ai" → Edition 输入 "Q3 2024 SaaS Benchmarks" → 通过年份选择器选择 "2024" → 输入 25th / Median / 75th 数值 → Data 选择 "Actual"
5. 点击 ✓ 确认保存
6. 重复步骤 3-5，录入 KeyBanc、High Alpha 平台的数据
7. 折叠该指标，处理下一个指标

**预期结果**：三个平台的最新数据均已录入，展开后可横向对比。

---

### 场景 2：跨平台数据对比

| 项目 | 说明 |
|------|------|
| **角色** | 投资经理 |
| **触发条件** | 评估被投企业某项指标是否达到行业基准 |
| **目标** | 快速获取多平台、多分位值的对比视图 |

**操作路径**：
1. 展开 "Gross Margin" 指标
2. 浏览来自不同平台的详情行
3. 对比同一年度下各平台的 25th、Median、75th 数值
4. 参考 "Best Guess" 字段中分析师的综合判断

**预期结果**：一屏内完成多平台基准数据对比，无需切换系统。

---

### 场景 3：历史数据修正

| 项目 | 说明 |
|------|------|
| **角色** | 数据管理员 |
| **触发条件** | 发现已录入数据存在错误 |
| **目标** | 快速修正或删除错误数据 |

**操作路径**：
1. 展开对应指标，定位到错误的详情行
2. 鼠标悬停该行 → 行尾出现 ✏️ 编辑图标和 🗑️ 删除图标
3. **修正**：点击 ✏️ → 修改错误字段 → 点击 ✓ 保存
4. **删除**：点击 🗑️ → 数据立即移除

**预期结果**：数据已修正或删除，无需二次确认弹窗，操作高效。

---

### 场景 4：公式录入

| 项目 | 说明 |
|------|------|
| **角色** | 高级分析师 |
| **触发条件** | 需要为指标记录计算公式，便于团队对齐口径 |
| **目标** | 在指标行中直接录入公式说明 |

**操作路径**：
1. 直接点击目标指标的 LG Formula 列
2. 输入公式，如 "(Current ARR - Previous ARR) / Previous ARR"
3. 内容实时保存

**预期结果**：公式即时生效，团队成员可直接查看。

## 三、功能模块

### 模块 1：顶部导航栏

提供全局上下文信息和系统级开关。

| 区域 | 元素 | 行为 |
|------|------|------|
| 左侧 | 公司 Logo（图片） | 纯展示，无点击行为 |
| 中右 | "Convert all currencies to USD" 文字 + Switch 开关 | 开关默认开启状态（`checked`）；开启时开关颜色为琥珀色 |
| 右侧 | 用户头像（圆形，首字母缩写） + 用户名 + 下拉箭头 | 纯展示，箭头暗示可扩展（当前无下拉菜单功能） |

---

### 模块 2：页面标题区

| 元素 | 内容 | 样式 |
|------|------|------|
| 主标题（H1） | "Benchmark Entry" | 28px，semibold，色值 #0D2B56 |
| 副标题 | "Manage and compare performance metrics across platforms" | 14px，灰色，Futura PT 字体 |

---

### 模块 3：指标分类管理（Looking Glass Category）

**定义**：Category 是固定的预定义分类，**不可增加、删除、重命名**。

**固定分类**（共 4 个）：

| 序号 | Category 名称 | 包含的 Metric 数量 |
|------|---------------|-------------------|
| 1 | Revenue & Growth | 1 |
| 2 | Profitability & Efficiency | 1 |
| 3 | Burn & Runway | 2 |
| 4 | Capital Efficiency | 2 |

**显示规则**：
- 同一 Category 下的所有 Metric 在表格中物理相邻
- Category 单元格使用 `rowSpan` 纵向合并，合并行数 = 该分类下所有 Metric 的行数之和（含展开的详情行和 "Add detail" 按钮行）
- Category 单元格垂直居中对齐，左对齐，带有右侧和底部边框，浅色背景

---

### 模块 4：指标管理（LG Metric）

**定义**：Metric 是固定的预定义指标，**不可增加、删除**。每个 Metric 隶属于唯一的 Category。

**固定指标**（共 6 个）：

| Category | Metric 名称 |
|----------|-------------|
| Revenue & Growth | ARR Growth Rate |
| Profitability & Efficiency | Gross Margin |
| Burn & Runway | Monthly Net Burn Rate |
| Burn & Runway | Monthly Runway |
| Capital Efficiency | Rule of 40 |
| Capital Efficiency | Sales Efficiency Ratio |

**交互行为**：

| 行为 | 说明 |
|------|------|
| 展开/折叠 | 点击 Metric 名称区域，切换详情数据的显示/隐藏；箭头图标同步切换（▶ ↔ ▼） |
| 折叠计数 | 折叠状态下，若存在详情数据，行尾显示 "{n} item(s)" 文字提示 |
| 公式编辑 | LG Formula 字段支持内联实时编辑（详见 4.5 节） |
| 多项展开 | 多个 Metric 可同时展开，互不影响 |

---

### 模块 5：详情数据管理（Detail Row）

**定义**：展开某个 Metric 后，可管理其下的详情数据行。每个 Metric 下可添加**不限数量**的详情行。

**支持操作**：

| 操作 | 触发方式 | 是否需确认 |
|------|----------|------------|
| 新增 | 点击 "Add detail" 按钮 | 需点击 ✓ 确认或按 Enter |
| 编辑 | 悬停后点击 ✏️ 编辑图标 | 需点击 ✓ 确认或按 Enter |
| 删除 | 悬停后点击 🗑️ 删除图标 | 无需确认，即时删除 |
| 取消 | 点击 ✕ 取消图标 | — |

## 四、操作流程与操作反馈

### 4.1 展开/折叠指标详情

```
用户点击 Metric 名称
    ├── 当前为折叠状态 → 展开
    │   ├── 箭头从 ▶ 变为 ▼
    │   ├── 该 Metric 下方显示所有详情行
    │   ├── 详情行下方显示 "Add detail" 按钮
    │   └── Category 单元格 rowSpan 动态增加
    └── 当前为展开状态 → 折叠
        ├── 箭头从 ▼ 变为 ▶
        ├── 详情区域隐藏
        ├── 若存在详情数据，行尾显示 "{n} item(s)"
        └── Category 单元格 rowSpan 动态减少
```

---

### 4.2 新增详情数据

```
前置条件：Metric 已展开

用户点击 "Add detail" 按钮
    │
    ├── 系统反馈：
    │   ├── 在详情列表末尾插入一行空白新增行
    │   ├── 新增行背景高亮（accent/30 色）
    │   ├── 各字段显示对应输入控件
    │   └── "Add detail" 按钮暂时隐藏
    │
    ├── 用户填写数据（所有字段均为非必填）
    │
    ├── 用户点击 ✓ 或按 Enter
    │   ├── 系统反馈：
    │   │   ├── 数据保存成功
    │   │   ├── 新增行消失
    │   │   ├── 数据以普通详情行样式出现在列表中
    │   │   ├── 所有字段重置为空（为下次新增准备）
    │   │   └── "Add detail" 按钮重新显示
    │
    └── 用户点击 ✕
        ├── 系统反馈：
        │   ├── 新增行消失
        │   ├── 所有已输入数据丢弃
        │   └── "Add detail" 按钮重新显示
```

---

### 4.3 编辑详情数据

```
前置条件：Metric 已展开，存在至少一条详情数据

用户鼠标悬停某条详情行
    │
    ├── 系统反馈：
    │   ├── 行背景变为悬停色
    │   └── 行尾渐变显示 ✏️ 编辑按钮和 🗑️ 删除按钮
    │
    ├── 用户点击 ✏️ 编辑按钮
    │   │
    │   ├── 系统反馈：
    │   │   ├── 该行所有字段从只读文本变为可编辑控件
    │   │   │   ├── 文本字段 → 输入框（Input）
    │   │   │   ├── Platform → 下拉选择（Select）
    │   │   │   ├── FY Period → 年份选择器（YearPicker）
    │   │   │   └── Data → 下拉选择（Select）
    │   │   └── 操作按钮变为 ✓（确认）和 ✕（取消）
    │   │
    │   ├── 用户修改字段 → 点击 ✓ 或按 Enter
    │   │   ├── 系统反馈：
    │   │   │   ├── 数据更新保存
    │   │   │   └── 行恢复为只读状态
    │   │
    │   └── 用户点击 ✕
    │       ├── 系统反馈：
    │       │   ├── 所有修改丢弃，恢复编辑前的值
    │       │   └── 行恢复为只读状态
```

---

### 4.4 删除详情数据

```
前置条件：Metric 已展开，存在至少一条详情数据

用户鼠标悬停某条详情行
    ├── 行尾出现 🗑️ 删除按钮（红色）
    │
    └── 用户点击 🗑️
        ├── 系统反馈：
        │   ├── 该行立即从列表中移除
        │   ├── 无二次确认弹窗
        │   ├── 不可撤销
        │   └── Category 单元格 rowSpan 自动调整
```

---

### 4.5 编辑 LG Formula

```
用户点击 LG Formula 列的输入框
    │
    ├── 系统反馈：
    │   ├── 输入框获得焦点
    │   └── 输入框边框从透明变为可见
    │
    ├── 用户输入/修改公式内容
    │   └── 系统反馈：内容实时保存，无需额外确认
    │
    └── 用户点击其他区域（失焦）
        └── 系统反馈：输入框边框恢复透明
```

**输入框状态变化**：

| 状态 | 边框 | 背景 |
|------|------|------|
| 默认 | 透明 | 透明 |
| 悬停 | 可见 | 透明 |
| 聚焦 | 可见（主色调） | 透明 |

---

### 4.6 货币转换开关

```
用户点击 Switch 开关
    ├── 开启 → 关闭：开关变为灰色
    └── 关闭 → 开启：开关变为琥珀色

备注：开关状态已持久化为 UI 状态，实际货币转换逻辑未实现。
```

### 6.3 字段输入规则

| 规则 | 说明 |
|------|------|
| 下拉选择字段 | 支持留空，不强制选择；选择后可通过编辑更换 |
| 自由文本字段 | 无字符数限制，支持中英文及特殊字符 |
| 数值字段（25th/Median/75th） | 建议输入数值或百分比，但不做格式强制校验，实际存储为字符串 |
| 年份字段 | 通过年份选择器组件选择，不支持手动键入；选择器支持十年段翻页 |
| Enter 快捷键 | 在文本输入框中按 Enter 等同于点击 ✓ 确认（新增和编辑场景均支持） |

| 属性 | 说明 |
|------|------|
| 触发方式 | 点击按钮弹出浮层（Popover） |
| 按钮内容 | 日历图标 + 已选年份（或占位文字 "Year"） |
| 按钮高度 | 32px |
| 浮层宽度 | 240px |
| 年份布局 | 3 列 × 4 行网格，显示 12 个年份 |
| 年份范围 | 当前十年段的前后各延伸 1 年（如 2019-2030） |
| 翻页 | 左右箭头按钮，每次切换 10 年（如 2020s → 2030s） |
| 当前年份 | 使用 accent 背景色高亮 |
| 已选年份 | 使用 primary 背景色高亮 |
| 十年外年份 | 半透明显示（opacity 50%） |
| 选择行为 | 点击年份后浮层自动关闭 |

## 七、业务规则

| 序号 | 规则名称 | 规则内容 |
|------|----------|----------|
| BR-01 | Category 不可变 | 4 个预定义分类固定不可增加、删除、重命名 |
| BR-02 | Metric 不可变 | 6 个预定义指标固定不可增加、删除；Metric 名称不可编辑 |
| BR-03 | Category-Metric 映射固定 | 每个 Metric 归属的 Category 固定，不可更改 |
| BR-04 | 分组相邻 | 同一 Category 下的 Metric 在表格中必须物理相邻 |
| BR-05 | 动态 rowSpan | Category 单元格的合并行数根据展开/折叠状态动态计算 |
| BR-06 | 唯一可编辑的父级字段 | 仅 LG Formula 可编辑，Category 和 Metric Name 均为只读 |
| BR-07 | 详情行无数量限制 | 每个 Metric 下可添加任意数量的详情行 |
| BR-08 | 展开状态独立 | 多个 Metric 可同时处于展开状态，互不干扰 |
| BR-09 | 悬停显示操作 | 编辑和删除按钮默认 opacity: 0，悬停时渐变到 opacity: 1 |
| BR-10 | 删除无确认 | 删除详情行即时生效，无二次确认弹窗，不可撤销 |
| BR-11 | 新增全字段可空 | 新增详情行时所有字段均为非必填，允许提交空行 |
| BR-12 | 折叠计数提示 | Metric 折叠时，若存在详情数据，显示 "{n} item(s)" |
| BR-13 | 数据前端持久化 | 当前数据仅存储在前端状态中，刷新页面后重置为初始样本数据 |
