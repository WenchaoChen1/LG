## 1. 功能概述

### 1.1 功能目的

Benchmark Entry 为投资分析团队提供一个统一的 SaaS 行业基准指标管理平台。它将分散在 Benchmarkit.ai、KeyBanc、High Alpha 等不同数据源的行业基准数据汇集到一个界面中，通过 Looking Glass 统一分类体系实现跨平台标准化对比，支持内联编辑快速录入，为投资决策提供多维度基准参考数据。

### 1.2 功能范围

**包含：**
- 基准指标的分类管理（Looking Glass Category）
- 指标的新增、编辑、展开/折叠
- 各平台详情数据的录入、编辑、删除
- 指标计算公式的录入
- 导航栏（Logo、货币转换开关 UI、用户信息）

**不包含（本期不做）：**
- 货币转换的实际计算逻辑（本期仅展示 UI 开关，不实现转换）
- 数据导入/导出功能
- 基准数据的报表分析或可视化图表
- 与 Benchmark 其他模块的数据联动（本功能仅负责数据录入，存入数据库供 Benchmark 模块使用）

### 1.3 相关角色

| 角色 | 与本功能的关系 |
|------|--------------|
| 投资分析师 | 主要使用者，负责日常基准数据的录入、更新和维护 |
| 投资经理 | 查看和对比各平台基准数据，用于投资决策参考 |
| 高级分析师 | 管理指标类别体系，新增类别和指标 |
| 数据管理员 | 维护历史数据，修正错误数据，清理过时数据 |

## 4. 业务流程

### 4.1 主流程：录入基准数据

```
步骤1：用户 进入 Benchmark Entry 页面
  └── 前置条件：用户已登录系统
  └── 执行结果：页面展示所有已有的类别和指标（默认折叠状态）

步骤2：用户 点击某个指标名称
  └── 前置条件：指标已存在
  └── 执行结果：该指标下方展开详情区域，显示已有详情数据行和 Add detail 按钮

步骤3：用户 点击 Add detail 按钮
  └── 前置条件：指标已展开
  └── 执行结果：在详情列表末尾显示一行空白新增行（带高亮背景），含 11 个输入字段和确认/取消按钮

步骤4：用户 依次填写字段（平台、版本、指标名、定义、年份、分段、分位值、数据类型、最佳猜测）
  └── 前置条件：新增行已显示
  └── 执行结果：字段值填入对应单元格

步骤5：用户 点击确认按钮（✓）或按 Enter 键
  ├── 提交成功 → 数据保存到数据库，新增行消失，数据出现在详情列表中
  └── 提交失败 → 显示错误提示（如网络异常），数据保留在新增行中

步骤6（可选）：用户 点击取消按钮（×）或按 Escape 键
  └── 执行结果：新增行消失，已填写的内容丢弃

终态：数据成功保存到数据库，在详情列表中可见
```

### 4.2 分支流程

**分支一：新建类别**
```
触发条件：表格中尚未添加所有 4 个预定义类别
步骤：
  1. 用户滚动到表格底部，点击 Add category 下拉菜单
  2. 系统展示尚未使用的类别选项
  3. 用户选择一个类别
  4. 系统自动创建该类别，并在其下生成该类别对应的第一个预定义指标
  5. 表格自动更新，新类别出现在表格末尾
结果：新类别及其默认指标出现在表格中
```

**分支二：新建指标**
```
触发条件：某类别下尚有未使用的预定义指标
步骤：
  1. 用户在目标类别组底部点击 Add metric 下拉菜单
  2. 系统展示该类别下尚未使用的指标选项
  3. 用户选择一个指标
  4. 系统在该类别组的最后一行之后插入新指标记录
结果：新指标出现在该类别组中；若该类别预定义指标已全部添加，Add metric 入口隐藏
```

**分支三：编辑类别/指标名称**
```
触发条件：用户需要修改已有的类别或指标名称
步骤：
  1. 用户将鼠标悬停在类别名称或指标名称上，出现编辑图标（铅笔）
  2. 点击编辑图标，文字变为输入框，同时显示确认和取消按钮
  3. 用户修改后按 Enter 或点击确认保存
  4. 若修改的是类别名称，系统自动同步更新所有属于该类别的记录
  5. 按 Escape 或点击取消放弃修改
结果：名称更新成功，页面即时刷新
```

**分支四：删除详情数据**
```
触发条件：用户需要移除某条详情数据
步骤：
  1. 用户将鼠标悬停在详情数据行上，行尾出现删除图标
  2. 点击删除图标
结果：该条数据立即从列表和数据库中移除，无二次确认
```

## 5. 业务规则

### 5.1 数据筛选与过滤规则

- **规则1**：Add category 下拉菜单仅展示尚未添加到表格中的预定义类别（适用场景：新建类别时）
- **规则2**：Add metric 下拉菜单仅展示该类别下尚未使用的预定义指标（适用场景：新建指标时）
- **规则3**：当某类别的所有预定义指标均已添加时，该类别组底部的 Add metric 入口隐藏（若类别仅有 1 个预定义指标，新建类别后该入口即刻隐藏）
- **规则4**：当所有 4 个预定义类别均已添加时，表格底部的 Add category 入口隐藏

### 5.2 状态与流转规则

本功能不涉及复杂状态流转。指标行仅有两种视觉状态：

```
折叠状态（默认）
  → 展开状态：用户点击指标名称

展开状态
  → 折叠状态：用户再次点击指标名称
```

多个指标可同时处于展开状态，互不影响。

### 5.3 权限与可见性规则

| 角色 | 可见范围 | 可操作项 | 限制说明 |
|------|---------|---------|---------|
| 投资分析师 | 全部类别、指标和详情数据 | 所有操作（新增/编辑/删除类别、指标和详情数据） | 无特殊限制 |
| 投资经理 | 全部类别、指标和详情数据 | 所有操作（新增/编辑/删除类别、指标和详情数据） | 无特殊限制 |
| 高级分析师 | 全部类别、指标和详情数据 | 所有操作（新增/编辑/删除类别、指标和详情数据） | 无特殊限制 |
| 数据管理员 | 全部类别、指标和详情数据 | 所有操作（新增/编辑/删除类别、指标和详情数据） | 无特殊限制 |

> 说明：本期所有角色的权限一致，均可执行所有操作。若后续需要差异化权限控制，需另行定义。

### 5.4 边界条件与异常规则

| 场景 | 处理规则 | 展示给用户的提示 |
|------|---------|----------------|
| 所有预定义类别已添加 | 隐藏 Add category 入口 | 无提示，入口不可见 |
| 某类别所有预定义指标已添加 | 隐藏该类别的 Add metric 入口 | 无提示，入口不可见 |
| 某指标下无详情数据 | 展开后显示空列表和 Add detail 按钮 | 无特殊空态提示，仅显示 Add detail |
| 网络异常导致新增保存失败 | 保留用户已填写的数据，不清空新增行 | 页面顶部 Toast 提示错误原因，3 秒后自动消失 |
| 内联编辑保存失败 | 恢复编辑前的值，不丢失原数据 | 页面顶部 Toast 提示错误原因，3 秒后自动消失 |
| 删除失败 | 数据行不移除，保持原状 | 页面顶部 Toast 提示错误原因，3 秒后自动消失 |
| 用户在新增行未填写任何字段直接确认 | 允许提交（所有详情字段均非必填） | 正常保存 |
| 已有未保存的新增行时再次点击 Add detail | 无响应，同一指标下同时仅允许一个新增行 | 无提示 |
| 类别重命名时输入空值 | 不允许提交，保持编辑状态 | 名称不能为空 |
| 指标重命名时输入空值 | 不允许提交，保持编辑状态 | 名称不能为空 |

| Edition | 数据版本/出版期号 | 纯文本 | — |
| Metric Name | 平台原始指标名称 | 纯文本 | — |
| Definition | 平台对该指标的定义 | 纯文本 | — |
| FY Period | 财年年份 | 年份数字（如 2024） | — |

| Best Guess | 分析师综合判断值 | 纯文本 | — |

**表格排序**：按类别添加顺序展示，类别内按指标添加顺序展示，详情行按添加时间正序排列。

| 触发操作 | 触发位置 | 目标页面/弹窗 | 说明 |

| 点击 × 按钮 / Escape | 新增行末尾 | 同页取消，新增行消失 | 已填内容丢弃 |
